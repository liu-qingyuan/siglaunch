# 域文档

此仓库采用单上下文布局。

## 探索规则

工程 skill 在探索代码库前应读取：

- 根目录的 `CONTEXT.md`
- `docs/adr/` 中与当前工作区域有关的 ADR

如果这些文件不存在，安静地继续。不要把缺失视为问题，也不要预先创建。
当术语或架构决策真正明确时，由 `/domain-modeling-lqy` 等 skill 按需创建。

## 布局

- `CONTEXT.md`：项目领域术语和统一语言
- `docs/adr/`：项目范围的架构决策记录

输出中的领域概念应使用 `CONTEXT.md` 定义的术语。若工作与现有 ADR
冲突，必须明确指出冲突，而不是默默覆盖原有决策。
